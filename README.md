# 코레일 작업현장 체감온도 AI 관리시스템

> 옥외 철도작업 현장의 **폭염·한파·호우 체감온도**와 **단계별 안전조치**를 한 화면에서 즉시 확인하는 모바일 웹(PWA).
> KORAIL 수도권서부본부 안전보건처 「AI 기반 체감온도 자동계산기」의 **전사(全社) 확대판**.

🔗 **라이브 데모:** https://korail-chegam-safety-2026.netlify.app
(현재 Mock 데이터로 동작 — 휴대폰 접속 시 실제 GPS 사용 가능. 단계 시연: 끝에 `?demo=danger`·`?demo=cold` 등)

## 핵심 기능

- **현 위치 / 전국 지역별 체감온도** — GPS 또는 전국 시·도→시·군·구→읍·면·동 선택
- **계절 통합** — 폭염(여름 체감온도)·한파(겨울 풍속 체감온도)·호우(특보·강수)
- **4단계 안전조치** — 정상·관심·주의보·경보 단계별 표준 조치사항 + 자동 팝업
- **체감온도 기록일지 자동화** — 법정 기록(31°C↑ 등) 자동 생성·저장·CSV/인쇄 내보내기
- **시간대별 예보 + 작업시간 권고** — 위험 시간대 회피 작업계획 지원
- **응급조치 가이드 + 현장 공유** — 온열·한랭질환 응급조치 + 링크/QR 전파
- **즐겨찾는 현장**, **PWA 오프라인**, **홈화면 추가**

## 기술 스택

- React 18 + Vite 5 + TypeScript + `vite-plugin-pwa`
- 기상청 공공데이터포털(초단기실황·예보) + Kakao Local — **서버리스 프록시**로 키 은닉
- **Mock 폴백**: API 키 없이도 전 기능 데모 가능

## 개발

```bash
npm install
npm run dev        # 개발 서버 (기본 Mock 데이터)
npm test           # 엔진 단위 테스트 (체감온도/단계/격자)
npm run build      # 타입체크 + 프로덕션 빌드
```

## 실데이터 연동 (환경변수)

서버리스 함수에 아래 키를 환경변수로 주입한다(코드에 커밋 금지):

| 변수 | 용도 |
|---|---|
| `KMA_SERVICE_KEY` | 기상청 공공데이터포털 서비스키 |
| `KAKAO_REST_KEY` | Kakao Local REST 키(역지오코딩) |

`.env.example` 참고. 키가 없으면 자동으로 Mock provider 로 동작한다.

## 배포

정적 프론트(`dist`) + 서버리스 함수(`netlify/functions`). `netlify.toml` 참고.
(사내망/타 호스팅 이식 가능 — 프록시는 표준 fetch 기반)

## 안전 관련 주의

- **폭염** 조치문구는 KORAIL 원본의 검증된 표준 문구.
- **한파·호우** 조치문구는 KOSHA·기상청 기준 **초안**이며, 정식 배포 전 **안전보건처 검토·확정 필요**.
